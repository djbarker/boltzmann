class boltzmann.utils.logger.PerfInfo(events: int = 0, micros: int = 0)[source]

Bases: object

Stores information about performance; namely a number of events and the time taken to run them.

PerfInfo objects can be added together (& subtracted) which is useful for measuring long-running performance.

events: int = 0

The number of events that processed.

micros: int = 0

The number of microseconds taken to process the events.

property seconds: float

The number of seconds taken to process the events.

property rate: float

The rate, measured in events/second.

fmt(events_name: str) → str[source]

A human-readable string formatted with the performance information.

perf_info = PerfInfo(2500, 1_250_000)
perf_info.fmt("frobinations") # == "2500 frobinations in 1.3 seconds => 2000.0 frobinations/sec"
Parameters:

events_name – The plural ‘name’ of the events being measured, e.g. “updates” or “frames”.

boltzmann.utils.logger.dotted(logger: Logger, label: str, value: Any, units: str = '', width: int = 40, level: int = 20)[source]

Log a value with some formatting to align consecuative calls.

dotted(logger, "x", 42)
dotted(logger, "pi", 3.14159)
dotted(logger, "g", 9.81, "m/s")

# output:
# x ................................... 42
# pi ............................. 3.14159
# g ................................. 9.81 m/s
Parameters:

  • logger – The logger to write the information to.

  • label – The label for the value.

  • value – The value to log.

  • units – The (optional) units for the value.

  • width – The width of the label and value (but not the units).

  • level – The log level to use (deault is logging.INFO).
