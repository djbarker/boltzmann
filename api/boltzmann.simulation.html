boltzmann.simulation

This module provides some “glue” for easily making scripts which run simulations. It provides automatic progress logging, checkpointing, and simulation resumption via the command line.

See guides/script for more information on how to use this module.

class boltzmann.simulation.IterInfo(interval: int, count: int)[source]

Bases: object

Contains the simulation output interval and count.

The output interval is the number of LBM iterations needed per output timestep. If one timestep of the LBM corresponds to dt and we wish to output every dt_output, then the number of LBM iterations per output is dt_output / dt.

interval: int

LBM iterations per output.

count: int

Total number of outputs.

static make(*, dt: float, dt_output: float | None = None, t_max: float | None = None, count: int | None = None) → IterInfo[source]

Construct a IterInfo object from a valid combination of the parameters. Valid combinations are those that are sufficient to infer the others.

Specific valid combinations are

  • dt, dt_output, count - interval is inferred as dt_output / dt.

  • dt, dt_output, t_max - count is inferred as t_max / dt_output, interval is inferred as above.

  • dt, t_max, count - interval is inferred as (t_max / count) / dt.

Parameters:

  • dt – The physical time corresponding to each iteration of the LBM simulation.

  • dt_output – The elapsed physical time after which we wish to produce each output.

  • t_max – The total physical time to run the simulation for.

  • count – The total number of outputs to produce.

boltzmann.simulation.parse_checkpoints(s: str) → CheckpointGater[source]

Parse a string into a CheckpointGater object.

There are only three valid formats:

  • "off" - which will disable checkpointing.

  • "<N>" - which will checkpoint every N iterations.

  • "<N>m"- which will checkpoint every N minutes.

boltzmann.simulation.run_sim(sim: Simulation, meta: IterInfo, output_dir: Path | str, checkpoints: CheckpointGater | None = None) → Generator[int, None, None][source]

Run the simulation with the given batch configuration, write checkpoints, and provide logging.

This returns a generator, which for each iteration runs the required number of LBM steps. It yields an integer which is the current iteration.

for i in run_sim(sim, meta, output_dir):
    plt.imshow(sim.fluid.rho.T)
    plt.savefig(output_dir / f"output_{i:04d}.png")

Once the generator resumes it outputs the checkpoint file in the specified directory with the name checkpoint.mpk. It is first written to a temporary file then moved into place to avoid possible corruption. The frequency of checkpointing is controlled by the checkpoints argument.

Parameters:

  • sim – The core Simulation object.

  • meta – The steps/output and output count info.

  • output_dir – The directory to write the checkpoint to.

  • checkpoints – The checkpointing frequency.

class boltzmann.simulation.SimulationScript(counts: ndarray | Sequence[int] | Sequence[float], omega_ns: float, meta: IterInfo, output_dir: str | Path, q: int | None = None)[source]

Bases: object

Allows us to easily build scripts that handle the initialization, output, running & checkpointing of the simulation.

This either constructs a new Simulation or loads one from the previously saved checkpoint, depending on the command line arguments passed.

  • In the case of constructing, the passed count and omega values are used, then the init function will be called to set the initial conditions.

  • In the case of loading, the passed arguments are ignored and the deserialized values are used. The init function is not called.

Under the hood it calls run_sim() so the behaviour is the same. However, this saves us worrying about manually loading checkpoints & wiring the command line arguments into the function calls.

from boltzmann.simulation import SimulationScript

# The simulation will run automatically when the with-block is exited.
# `sim` is a Simulation object.
with (script := SimulationScript(...)) as sim:

    @script.init
    def init():
        # Set some initial conditions.
        pass

    @script.out
    def out(output_dir: Path, iter: int):
        # Save some output.
        pass
