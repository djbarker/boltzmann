boltzmann.core module

The core lattice Boltzmann simulation classes.

The Rust extension module is re-exported here for convenience & clarity. If you wish to import from that directly you can import boltzmann.boltzmann.

See guides/basic for more information on how to use this module.

class boltzmann.core.CellFlags[source]

Bases: object

Flags which control the simulations behaviour at a given cell. Multiple flags can be combined using the bitwise OR operator.

FLUID = 0

The default value, the fluid & any tracers will evolve as usual.

WALL = 1

The fluid obeys the no-slip boundary condition & tracers will see zero concentration gradient.

FIXED_FLUID_VELOCITY = 2

The fluid velocity is fixed at the initial value, the density will evolve as usual.

FIXED_FLUID_DENSITY = 4

The fluid density is fixed at the initial value, the velocity will evolve as usual.

FIXED_SCALAR_VALUE = 8

The scalar value is fixed at the initial value. The fluid evolves as normal.

FIXED_FLUID = 6

Both fluid density & velocity are fixed to their initial values.

boltzmann.core.calc_lbm_params_si(dx: float, u_si: float, L_si: float, nu_si: float, tau: float | None = None, M_max: float = 0.1, tau_max: float = 1.0, tau_min: float = 0.5, slack: float = 0.0) → tuple[float, float][source]

Choose the values of tau and u which maximize simulation speed & verify stability. If you specify tau that value will be used instead.

Sometimes it is more natural to specify the physical parameters directly rather than via Reynold’s number. Under the hood, however, this function calls calc_lbm_params_lu().

Parameters:

  • dx – Lattice spacing.

  • u_si – Characteristic velocity of the system.

  • L_si – Characteristic length-scale of the system.

  • nu_si – Kinematic viscosity of the system.

  • tau – Relaxation time of the BGK collision operator.

  • M_max – Maximum Mach number of the system.

  • tau_max – Hard cap on the value of tau allowed.

  • tau_min – Hard floor on the value of tau allowed.

  • slack – See check_lbm_params().

boltzmann.core.check_lbm_params(Re: float, L: float, tau: float, M_max: float = 0.1, slack: float = 0.0)[source]

Check if the chosen parameters are likely to be stable or not with BGK collision operator. For more information see https://dbarker.uk/lbm_parameterization/.

Parameters:

  • Re – Desired Reynold’s number of the system.

  • L – Characteristic length of the system.

  • tau – Relaxation time of the system.

  • M_max – Maximum Mach number of the system.

  • slack – Extra margin on the checks. Must be in range (-1, inf). Values < 0 imply stricter checks, wheras > 0 imply more relaxed checks.

add_tracer(name, omega_ad, q=None)

Add a scalar field which will follow the advection-diffusion equation.

Returns the Scalar object for the added tracer; you must keep this around to access the data. This function is idempotent (over the name) and will return the existing tracer if it already exists.

Parameters:

  • name – The unique name of the tracer.

  • omega_ad – The relaxation parameter for the advection-diffusion equation.

  • q – The number of discrete velocities to use for the advection-diffusion equation kernel.

Returns:

The Scalar object.
